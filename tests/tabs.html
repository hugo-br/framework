General instructions

  • Try to keep image size under 1000KB (if possible) as it's longer to load on the website
  • Try to do a slide per link for landing page
  • Do not put CTA on desktop images. Put them on the mobile.
  • When you do changes, keep the same name as the previous or add '-v2' at the end. Do not rename them.
  • When doing a new page, don't name them as the same of the previous one, change the date.
  • How to name pictures :
    • Try to follow the naming convention on each components as it's easier for the changes
    • Privilege '-' instead of '_' on the pciture names, it's easier to read and less prone to mistakes
    • Keep the same name for each language and devices, only change the locale on the picture and
    • Use locale (-en) and (-fr). If there's US version use same name with (-us) instead of the '-en-'
  • Try to minimize the use of gifs (usually too heavy for the effects wanted) and image-maps (longer to do, not always responsive friendly)
  • Do not put margins on the slides. Except if the CTA is at the bottom of the page, then please add a little white space, usually the CTA height +/- 50px.
  • Before sending for coding, make sure that every pictures has been proofed, and it follow the right format and naming convention.
  • Every components on this page can be added to the landing page, banners, contests pages, forms, etc...
  • Customize or new components can be made within 3 to 5 working days (depending on the complexity) after resquested with no exception
